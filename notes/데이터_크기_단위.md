# 컴퓨터 데이터 크기 단위 정리

## 1. 비트(bit) — 가장 작은 단위

- 컴퓨터가 다루는 **가장 작은 단위**
- 값은 딱 두 가지: **0 또는 1** (전기 꺼짐/켜짐)
- 비트 n개로 표현 가능한 가짓수 = **2ⁿ 가지**

```
비트 1개 → 0, 1              (2가지)
비트 2개 → 00 01 10 11        (4가지)
비트 3개 → 000 ... 111        (8가지)
```

## 2. 바이트(byte) — 기본 단위

- **1바이트(byte) = 8비트(bit)**
- 컴퓨터가 데이터를 다루는 **기본 묶음 단위** (메모리 한 칸 = 보통 1바이트)
- 표현 가능한 가짓수 = 2⁸ = **256가지** (0~255)
- 영어 글자 하나(A, b, 7 등)가 보통 1바이트

```
1 byte = 8 bit
┌─┬─┬─┬─┬─┬─┬─┬─┐
│0│1│1│0│1│0│0│1│
└─┴─┴─┴─┴─┴─┴─┴─┘
```

## 3. 큰 단위들 (1024씩 커짐)

| 단위 | 읽기 | 크기 | 대략 |
|------|------|------|------|
| **B**  | 바이트 | 8 bit | 글자 1개 |
| **KB** | 킬로바이트 | 1,024 B | 짧은 문서 |
| **MB** | 메가바이트 | 1,024 KB | 사진 1장, MP3 |
| **GB** | 기가바이트 | 1,024 MB | 영화 1편 |
| **TB** | 테라바이트 | 1,024 GB | 하드디스크 |
| **PB** | 페타바이트 | 1,024 TB | 대형 데이터센터 |

```
bit → byte → KB → MB → GB → TB → PB
 ×8    ×1024  ×1024 ×1024 ×1024 ×1024
```

### 왜 1024일까? (1000이 아니라)

- 컴퓨터는 **2진수(2의 거듭제곱)** 를 사용
- 2¹⁰ = 1024 ← 1000에 가장 가까운 2의 거듭제곱
- ⚠️ 엄밀히는 KiB(키비바이트)=1024, KB=1000으로 구분하기도 함

## 4. 자바 자료형 크기

| 자료형 | 크기 | 비트 |
|--------|------|------|
| `byte`    | 1 byte | 8 bit |
| `short`   | 2 byte | 16 bit |
| `int`     | 4 byte | 32 bit |
| `long`    | 8 byte | 64 bit |
| `float`   | 4 byte | 32 bit |
| `double`  | 8 byte | 64 bit |
| `char`    | 2 byte | 16 bit |
| `boolean` | 1 byte* | (논리상 1bit) |

> 예) `int`가 4바이트(32비트) → 메모리 칸 4개 차지 → 표현 범위 2³² 가지(약 -21억 ~ +21억)

## 5. 왜 8비트가 1바이트일까? (역사)

처음부터 8은 아니었음 (초기엔 6·7·9비트 제각각). 여러 이유가 겹쳐 8로 굳어짐.

| 이유 | 내용 |
|------|------|
| 📝 문자 표현 | 영어 글자+기호 다 담으려면 7~8비트 필요 (ASCII는 7비트=128가지) |
| 🔢 2의 거듭제곱 | 어중간한 7보다 깔끔한 8이 회로 설계에 유리 |
| 🏢 IBM 표준화 | 1964년 IBM System/360이 8비트 채택 → 업계 표준이 됨 |
| ✂️ 나누기 편함 | 8 = 4+4(니블), 4비트는 16진수 한 자리(0~F)에 딱 맞음 |

```
8비트 = 1111 0000  →  16진수 "F0" 으로 깔끔하게 표기
```

> 한 줄 요약: **영어 글자엔 7비트면 충분했지만, 2의 거듭제곱이라 다루기 깔끔하고 IBM이 표준으로 정한 8비트가 살아남았다.**

## 핵심 흐름 요약

```
비트(bit)      ← 가장 작은 단위 (0 또는 1)
  ↓ ×8
바이트(byte)   ← 기본 단위 (글자 1개)
  ↓ ×1024
KB → MB → GB → TB → PB ...
```
